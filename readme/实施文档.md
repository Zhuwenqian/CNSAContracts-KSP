# 实施文档

## 1. 安装方法
1. 确保已安装 KSP 1.8.1–1.12.x 版本。
2. 安装依赖模组：
   - Contract Configurator
   - Real Solar System (RSS)
   - Realism Overhaul (RO)
   - KerwisChineseAerospacePack（提供 DFH-1 与 CZ-1 部件）
3. 将本仓库中的 `GameData/CNSAContracts/` 文件夹复制到 KSP 安装目录的 `GameData/` 下。
4. （可选）为机构准备标志贴图：
   - 256×160 尺寸：`GameData/CNSAContracts/Agencies/DFH1_Agency.png`
   - 64×40 尺寸：`GameData/CNSAContracts/Agencies/DFH1_Agency_scaled.png`
   - 若暂不提供贴图，KSP 将使用默认机构图标。
5. 启动游戏，在主菜单按 `Alt+F10` 打开 Contract Configurator 调试窗口，确认合同无红色报错。

## 2. 游戏内效果
- 游戏开始后，当玩家满足以下条件时，任务控制中心会提供“东方红一号卫星发射任务”：
  - 已解锁 DFH-1、CZ1_stage1、CZ1_stage2、CZ1_stage3、CZ1_coupling、CZ1_fairing 部件；
  - 已达到过太空（ReachSpace）；
  - 当前 RO 时间早于 1970 年 4 月 30 日。
- 合同要求：
  - 接受合同后发射一枚新火箭；
  - 火箭发射时必须同时包含 CZ-1 一级、二级、三级、级间段、整流罩以及 DFH-1 卫星部件；
  - 进入地球轨道，且近地点 ≥ 140 km、远地点 ≤ 2000 km。
- 合同截止时间显示为 1970 年 4 月 30 日；超过该日期后不再提供新合同。
- 因 `VesselParameterGroup` 设置 `completeInSequence = true`，CZ-1 各级部件在火箭刚发射、尚未分离时即被检测并锁定，入轨后分离掉的部件不会导致参数失败。

## 3. 常见问题与调整

### 3.1 起始年份不是 1951
若使用的 RP-1/RSS 配置将生涯起点设为其他年份，需重新计算 `targetDateSeconds`：
1. 计算从起点到 1970-04-30 的总天数（注意闰年）。
2. 总秒数 = 总天数 × 86400。
3. 修改 `GameData/CNSAContracts/Contracts/DFH1_Launch.cfg` 中 `DATA/targetDateSeconds` 的值。

### 3.2 不想强制使用 CZ-1 部件
可删除或注释掉以下节点：
- `REQUIREMENT` 中的 `PartUnlocked_CZ1_*`；
- `PARAMETER` 中的 `PartValidation_CZ1_*`。
但这样会削弱任务的历史还原度。

### 3.3 不想强制使用 DFH-1 部件
可删除或注释掉以下节点：
- `REQUIREMENT` 中的 `PartUnlocked_DFH1`；
- `PARAMETER` 中的 `PartValidation_DFH1`。

### 3.4 调整奖励
在 `DFH1_Launch.cfg` 中修改以下字段：
- `advanceFunds`：接受合同时预付款；
- `rewardFunds`：完成任务奖励资金；
- `rewardScience`：奖励科技点；
- `rewardReputation`：奖励声望。
