# 开发文档

## 1. 项目目标
为 RSS-RO 环境开发一个基于 Contract Configurator 的中国航天历史任务合同包。首个任务为东方红一号卫星发射任务。

## 2. 依赖模组
- **Contract Configurator**：提供 CONTRACT_TYPE / CONTRACT_GROUP / AGENT 等配置节点。
- **Real Solar System (RSS)**：将 Kerbin 重命名为 Earth，并将游戏起点设为 1951-01-01 TAI。
- **Realism Overhaul (RO)**：提供真实化的部件与生涯环境。
- **KerwisChineseAerospacePack**：提供以下部件：
  - DFH-1 卫星（name = DFH-1）
  - CZ1_stage1 一级燃料箱（name = CZ1_stage1）
  - CZ1_stage2 二级箭体（name = CZ1_stage2）
  - CZ1_stage3 三级固体发动机（name = CZ1_stage3）
  - CZ1_coupling 级间段/连接环（name = CZ1_coupling）
  - CZ1_fairing 整流罩（name = CZ1_fairing）

## 3. 目录与文件设计
```
GameData/CNSAContracts/
├── Agencies/
│   └── Agents.cfg          # 定义专属机构
├── Groups.cfg              # 定义合同组 CNSA_HistoricalMissions
└── Contracts/
    └── DFH1_Launch.cfg     # 东方红一号发射任务合同
```

## 4. 核心设计决策

### 4.1 机构独占
- 使用 `AGENT` 节点定义“中华人民共和国国防部第五研究院”。
- 设置 `mentality = NoRandomContractMentality`，禁止 KSP 随机合同系统占用该机构。
- 合同组与合同均通过 `agent` 字段显式引用该机构。

### 4.2 截止时间实现
- Contract Configurator 的 `deadline` 字段本身为相对时长（接受合同后计时），不支持绝对日历日期。
- 本包采用 `DATA` 节点存储目标秒数 `targetDateSeconds = 609984000.0`（对应 1970-04-30 00:00:00）。
- `deadline` 与 `maxExpiry` 使用表达式 `Duration(@/targetDateSeconds - UniversalTime())`，在合同生成时动态计算剩余时间。
- 同时增加 `Expression` 要求 `UniversalTime() < @/targetDateSeconds`，确保截止日期后不再提供新合同。
- 注意：动态截止时间在合同生成时求值，建议玩家尽快接受以获得最接近真实历史的截止日期。

### 4.3 轨道判定
- 使用 `Orbit` 参数：
  - `minPeA = 140000`（近地点 ≥ 140 km）
  - `maxApA = 2000000`（远地点 ≤ 2000 km）
- 偏心率与倾角不限制，允许椭圆轨道。

### 4.4 部件验证
- 使用 `PartUnlocked` 要求玩家已解锁 DFH-1 与全部 CZ-1 相关部件。
- 使用 `VesselParameterGroup` 组合多个 `PartValidation` 参数，要求运载火箭同时包含：
  - CZ1_stage1、CZ1_stage2、CZ1_stage3、CZ1_coupling、CZ1_fairing
  - DFH-1 卫星部件
- 设置 `completeInSequence = true`，使 CZ-1 各部件在火箭刚发射、尚未分离时即被检测并锁定，避免入轨后分离掉的级间段/整流罩导致参数失败。
- 使用 `NewVessel` 参数确保飞船为接受合同后新发射。

## 5. 扩展计划
- 未来可在 `Contracts/` 目录下继续添加新的历史任务合同。
- 新合同统一归属 `CNSA_HistoricalMissions` 组，共享机构与并发控制。
